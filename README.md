# Agent Arena

**The open-source arcade intelligence layer for the AI agent race.**

Agent Arena starts as a live Claude vs Codex dashboard, but the bigger idea is simple: turn the AI agent race into a public, remixable game that anyone can extend with any coding agent.

Find the repo. Like the idea. Point Codex, Claude Code, Cursor, Devin, Replit, v0, Lovable, or any other builder-agent at it. Ship a weird, useful, funny, ambitious PR. Repeat until the platform becomes something the community could not have planned from the top down.

## The Loop

1. Fork or clone the repo.
2. Ask your coding agent to understand the app.
3. Give it one direction: make Agent Arena more addictive, useful, beautiful, or commercially interesting.
4. Review the diff like an owner.
5. Open a PR with screenshots, notes, and which agent helped.

Copy-paste prompt:

```text
You are contributing to Agent Arena, an open-source arcade intelligence platform for the AI agent race.

Read the README and codebase. Pick one focused improvement that makes the platform more viral, useful, beautiful, monetizable, or community-driven. Implement it, run the available checks, and prepare a PR summary with screenshots or verification notes.
```

## What You Can Build

Bring the platform in any direction that makes the arena more alive:

- **New game modes:** leagues, seasons, quests, factions, tournaments, streaks, unlocks.
- **Better intel:** richer public-source pipelines, launch tracking, agent release timelines, benchmark feeds.
- **UGC:** let builders, founders, researchers, and key people enrich their own characters or pages.
- **Prediction layers:** market-style forecasts, Polymarket integrations, community odds, agent matchup probabilities.
- **Monetization:** sponsor slots, premium analytics, team pages, paid APIs, pro dashboards.
- **Mobile:** iOS and Android apps, widgets, push alerts, watchable daily recaps.
- **Token experiments:** reputation, contribution points, creator economies, community governance.
- **Social loops:** share cards, daily battles, leaderboards, X/LinkedIn snippets, embeddable widgets.
- **Visual systems:** better sprites, arena maps, character cards, motion, sound, skins, lore.
- **Infrastructure:** tests, data contracts, moderation tools, analytics, deployment hardening.

Small PRs are welcome. Wild PRs are welcome. Design-only, data-only, copy-only, and agent-generated PRs are welcome if they make the project better.

## Contribution Rules

- Keep PRs focused enough to review.
- Include screenshots for UI changes.
- Run the checks you can and mention what you skipped.
- Do not commit secrets, private data, or generated dependency folders.
- Use public sources for claims about real companies or people.
- Avoid harassment, impersonation, defamatory claims, or private personal information.
- Disclose the coding agent or tools that helped, if any.

This project should be built in public, but it should stay fun, useful, and credible.

## Intel Pipeline

The app reads `data/intel/current-snapshot.json`, generated by the local pipeline. Static data in `data.js` remains as a fallback if the snapshot is malformed.

```bash
npm run intel:offline   # build from seeded official-source backfill
npm run intel:build     # use X API and xAI when credentials are configured
```

Configure credentials in the repo-local `.env`:

```bash
X_BEARER_TOKEN=...
XAI_API_KEY=...
```

The daily run writes `data/intel/daily/YYYY-MM-DD.json` and updates `data/intel/current-snapshot.json`.
